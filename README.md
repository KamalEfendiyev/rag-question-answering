# rag-question-answering
rag-with-reranking A retrieval-augmented QA system with reranking and evaluation
# RAG с реранкингом: система вопрос-ответ

В проекте реализован полный pipeline Retrieval-Augmented Generation (RAG) для задачи question answering: от поиска документов до генерации ответа с помощью LLM.

---

## 🧠 Архитектура системы

- Запрос  
- ↓  
- Retrieval (Dense + BM25)  
- ↓  
- Топ-k документов  
- ↓  
- Reranking (cross-encoder)  
- ↓  
- Лучший контекст  
- ↓  
- LLM (FLAN-T5)  
- ↓  
- Ответ

---


## 📁 Структура проекта

### 📥 Загрузка и подготовка данных
Загрузка датасета SQuAD, извлечение вопросов (`queries`) и уникальных контекстов (`documents`), используемых для поиска.

---

### 🔍 BM25 Retrieval (sparse поиск)
Реализация классического поиска по ключевым словам с использованием BM25.  
Используется как baseline для сравнения с нейронными методами.

---

### 🧠 Dense Retrieval (Embeddings + FAISS)
Кодирование документов с помощью SentenceTransformers и построение FAISS-индекса для быстрого поиска по эмбеддингам.  
Позволяет находить документы по смыслу, а не только по совпадению слов.

---

### 📊 Оценка Retrieval (Recall@5)
Оценка качества поиска с помощью метрики Recall@5.  
Показывает, насколько часто релевантный документ попадает в топ-k результатов.

---

### 🔄 Reranking (Cross-Encoder + MRR)
Реранкинг найденных документов с помощью cross-encoder модели.  
Оценка качества выполняется через MRR (Mean Reciprocal Rank), отражающую позицию правильного документа.

---

### 🤖 RAG (Retrieval-Augmented Generation)
Полный pipeline: retrieval → reranking → генерация ответа с помощью LLM (FLAN-T5).  
Модель генерирует ответ только на основе найденного контекста.

---

### 📊 Оценка RAG
Оценка качества ответов с помощью метрик:
- Overall Accuracy  
- Answered Accuracy  
- Answer Rate  

Позволяет анализировать баланс между точностью и покрытием.

## ⚙️ Компоненты

### 1. Retrieval

- Dense retrieval: `sentence-transformers (all-MiniLM-L6-v2)`
- BM25: классический поиск по словам
- Hybrid retrieval: объединение Dense и BM25

Задача этапа: максимизировать **recall** (не потерять релевантные документы)

---

### 2. Реранкинг

- Модель: `cross-encoder/ms-marco-MiniLM-L-6-v2`
- Оценивает пары (запрос, документ)

Задача этапа: улучшить **порядок документов** (релевантный документ на 1 месте)

---

### 3. Генерация ответа (RAG)

- Модель: `FLAN-T5`
- Генерация ответа на основе найденного контекста

Используется ограничение:

> Ответ должен быть получен только из контекста.  
> Если ответа нет — модель должна вернуть "I don't know".

---

## 📊 Метрики

### Retrieval

- Recall@k
- MRR (Mean Reciprocal Rank)

Пример:
- Dense MRR: 0.75
- Rerank MRR: 0.80


Реранкинг улучшает позицию релевантных документов.

---

### RAG

Используются три метрики:

- **Overall Accuracy** — доля правильных ответов от всех запросов
- **Answered Accuracy** — точность только по данным ответам
- **Answer Rate** — доля случаев, когда модель даёт ответ

Пример:


- Overall Accuracy: 0.55
- Answered Accuracy: 0.69
- Answer Rate: 0.80


---

## 🔍 Наблюдения

- Увеличение количества документов в контексте (top-k) может ухудшать качество из-за шума
- Реранкинг стабильно улучшает качество ранжирования
- Основное узкое место — качество retrieval
- Ограничение "I don't know" снижает количество ошибок, но уменьшает покрытие

---

## ⚖️ Компромиссы

| Параметр | Эффект |
|--------|--------|
| Увеличение top-k | ↑ recall, ↓ качество ответа |
| Реранкинг | ↑ качество ранжирования |
| Ограничение ответа | ↓ галлюцинации, ↓ answer rate |

---

## 🚀 Возможные улучшения
